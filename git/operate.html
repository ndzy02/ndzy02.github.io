# basics 基本操作

git init <directory> 初始化仓库 eg: git init 将当前文件夹初始化为 git 仓库

git clone <repo> 克隆项目

git config user.name name 设置开发人员名称

git add <directory>|<file> 暂存文件

git commit -m "<message>" 提交暂存

git status

# git log

  • git log -<limit> eg: git log -1 展示最近一条提交信息
  • git log --oneline 将每个提交压缩为一行展示
  • git log -p 显示每个提交的完整差异
  • git log --stat 显示简略提交信息
  • git log --author <pattern> eg: git log --author zhangyi
  • git log --grep <pattern> 搜索提交记录 eg: git log --grep chore
  • git log -- <file> 文件提交记录

# git reflog 显示本地存储库 HEAD 的更改记录

# git diff

  • git diff 工作区与暂存区比较
  • git diff --cached <path> 暂存区与最近一次提交比较
  • git diff HEAD <path> 工作区与最近一次提交比较
  • git diff commit-id <path> 工作区与指定提交比较
  • git diff --cached commit-id <path> 暂存区与指定提交比较
  • git diff commit-id1 commit-id2 两次提交比较
  • git diff > <patch> 打补丁 eg: git diff --cached > diff.txt 暂存区与版本库的差异做成补丁输出到文件 diff.txt

# undoing changes 撤消更改

# git revert 撤销

git revert 用一次新的提交来回滚之前的提交，版本会递增，不影响之前提交的内容

  • git revert HEAD 撤销前一次提交
  • git revert HEAD^ 撤销前前一次提交
  • git revert commit-id 撤销指定版本,撤销也会作为一次提交保存

# git reset 版本回退

  • git reset --hard HEAD|commit-id 版本回退 并删除版本内容
  • git reset --soft HEAD|commit-id 版本回退 版本内容回退到暂存区
  • git reset HEAD|commit-id 版本回退 版本内容回退到工作区

# git clean

批量删除 branch 中新加的文件(untracked files) 未跟踪的文件 ; 删除前 加上 命令 n 防止误删除

  • n
  • f 文件
  • d 目录
  • x gitignore 的文件

# git commit --amend 编辑最后一次提交

git commit --amend --no-edit 编辑最后一次提交信息

适用场景:修改了一个笔误或添加了一个忘记暂存的文件 (将需要更改或新增的文件加入暂存区即可)

# git rebase （不要合并已提交到远程分支的 commit）

功能:

  • 合并 commit
  • 将某一段 commit 粘贴到另一个分支上

git rebase -i HEAD~4 合并最近四次提交

  • pick：保留该 commit（缩写:p）

  • reword：保留该 commit，但我需要修改该 commit 的注释（缩写:r）

  • edit：保留该 commit, 但我要停下来修改该提交(不仅仅修改注释)（缩写:e）

  • squash：将该 commit 和前一个 commit 合并（缩写:s）

  • fixup：将该 commit 和前一个 commit 合并，但我不要保留该提交的注释信息（缩写:f）

  • exec：执行 shell 命令（缩写:x）

  • drop：我要丢弃该 commit（缩写:d）

  • 异常退出 git rebase --edit-todo 恢复

  • git rebase --continue
